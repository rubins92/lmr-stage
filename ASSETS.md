# Видео-ассеты лендинга

Видео (`*.mp4`, `*.mov`, `*.webm`) **намеренно не лежат в git** — бинарники раздувают историю репозитория навсегда (не дельтятся), а у нас командный репо. Картинки (`*.png`, `*.svg`, `og-image.jpg`) — в git, они мелкие и нужны для превью.

## Если ты только что клонировал репо

Локальное превью (`index.html`) ссылается на видео в `assets/`. После чистого клона их не будет — видео нужно положить вручную:

- `assets/lumir-promo.mp4` — промо на главном экране (hero)
- `assets/flow-approve.mp4`, `assets/flow-photo.mp4`, `assets/flow-psd.mp4` — анимации шагов

**Где взять:** попроси у Сергея папку с видео, либо вытяни актуальные из задеплоенного репо:

```bash
# из GitLab dev/lumir-site (там лежат уже оптимизированные под веб версии)
# (нужен доступ к gitlab.lumir.photo)
```

Без видео страница работает, но hero и блоки шагов покажут пустые плееры.

## Если добавляешь / меняешь видео

1. Положи файл в `assets/` (например `assets/lumir-promo.mp4`).
2. Деплой сам пережмёт его под веб (`optimize-video.sh`: faststart + разумный битрейт) и зальёт на прод. Ничего вручную кодировать не надо.
3. Git его проигнорирует — это нормально, так задумано.

## Почему так

Сырой экспорт из видеоредактора обычно идёт с `moov`-атомом в конце файла → браузер не может начать воспроизведение, пока не скачает почти весь файл. Выглядит как «видео не грузится / тормозит». Пайплайн деплоя автоматически чинит это (`-movflags +faststart`) и ужимает размер, так что на `lumir.photo` всегда едет веб-готовая версия.
