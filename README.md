# lmr-stage — Lumir Landing Staging

Совместный стейджинг лендинга **lumir.photo**. Редактируем здесь через Claude Code, апрувленные правки Сергей переносит в прод вручную.

## Что внутри

- `index.html` — единственная страница лендинга
- ассеты: `favicon.*`, `hero-banner-*.png`, `logo-lumir-photo.png`, `flow-*.mp4`
- `.claude/CLAUDE.md` — инструкции для Claude Code

## Стек

Чистый **HTML + Tailwind CDN + ванильный JS**. Никакой сборки, никакого npm. Один файл `index.html` с inline-стилями и скриптами.

## Как работать (3 шага)

### 1. Клонируешь

```bash
git clone https://github.com/rubins92/lmr-stage.git
cd lmr-stage
```

### 2. Запускаешь Claude Code и поднимаешь превью

```bash
claude
```

В Claude Code попроси:
> Запусти превью на localhost

Или сам в терминале:
```bash
npx serve .
```
→ открой `http://localhost:3000` в браузере. Превью обновляется при перезагрузке страницы.

### 3. Правишь и пушишь

```bash
git add .
git commit -m "коротко что поменял"
git push
```

Сергей увидит изменения в GitHub, ревьюнет и при апруве перенесёт в прод.

## Конвенции

- **Коммит-сообщения** — кратко на русском или английском, по делу. Примеры: `update hero copy`, `fix mobile spacing section 3`, `replace section-2 image`.
- **Текст на лендинге — только русский.**
- **Ассеты** — клади в корень папки, как `hero-banner-final.png`. Без подпапок.
- **Большие видео (>20 МБ)** — НЕ пушь. Пришли Сергею отдельно.
- **Не трогай** `.claude/CLAUDE.md` — это инструкции для Claude Code.

## Что делать НЕ нужно

- Не делай Pull Request — пушь прямо в `main`. Доверяем друг другу, ревью делает Сергей перед прод-деплоем.
- Не настраивай CI / GitHub Actions — деплой ручной у Сергея.
- Не удаляй файлы без согласования.

## Если сломал — откатись

```bash
git log --oneline -10           # найди коммит до поломки
git revert <sha>                # откатить один коммит
git push
```

Или скажи Сергею — он откатит.
